v0.3.0 — Discoverability sweep + Boolean type

The first release where opening the app for the first time and
actually finding the power-user features is plausible. The P6
"Discoverability + help system overhaul" arc landed in full
across calculator history, notepad lines, both keypad tabs
(Adv + CAS), and all eight Analyze-hub module screens — each
surface now answers "what is this?" with one tap.

Headline changes since v0.2.3:

**Discoverability — P6 (rounds 93–105)**

- **Worked Examples library** discoverable from Calculator +
  Notepad (rounds 93, 94, 95). The library opens the right
  module pre-loaded for cross-module examples (Sudoku-X,
  Welch t-test, CSP DSL, etc.).
- **Function Reference dialog** with 45 catalogued entries
  (rounds 96, 96b, 97, 98, 99) covering CAS, the
  precision arc, number theory, matrix / linear algebra,
  statistics, constraints, sudoku.
- **Help-mode toggle** in Calculator + Notepad AppBars
  (round 101). When on, every help-aware control draws a
  dotted blue outline so users see what's interactive.
- **Help popovers on Calculator keypad** — Adv tab (round
  102) + CAS tab (round 102b). Tapping a button in help mode
  opens its function-reference card with a deep-link into
  the full dialog.
- **Help popovers on history rows + notepad lines** (rounds
  103, 104, 104b). The popover explains how the row was
  computed — SymEngine, MPFR, FLINT, Dart — and, for solve /
  diff / integrate, re-runs the step-by-step trace dialog
  on demand.
- **Module help on Analyze hub** (round 105). Each of the
  eight module screens (curve sketching, planes, conics,
  statistics, 3D graphing, 3D scene, constraints, sudoku)
  has a `(?)` AppBar button opening a 2–3 sentence
  explainer with optional deep-link.

**Boolean type — P7 (rounds 110–113)**

- `==`, `≠`, `<`, `≤`, `>`, `≥`, `and`, `or`, `not`, `xor`
  on the Adv keypad with worked examples (round 112).
- `if(cond, then, else)` with Dart-side folding (round
  111b) for conditional arithmetic.
- Notepad result column renders boolean results as a
  colored chip via the shared `BooleanChip` widget (round
  113).

**Test coverage**: 1992 tests pass at v0.3.0 (up from
~1700+ at v0.2.3). `flutter analyze` clean. The two new
top-level test files: `history_help_modal_test.dart`,
`module_help_dialog_test.dart`.

**Platform support — SymEngine bridge availability**

Unchanged from v0.2.3:

- **macOS, iOS** — full symbolic + numeric features.
- **Linux, Windows, Android** — degraded mode. The bundled
  `flutter_symengine_*` plugin ships compiled libs for
  iOS/macOS only; on other platforms symbolic ops (`solve`,
  `factor`, `expand`, `simplify`, `d/dx`, `integrate`,
  `limit`) return error strings. Help-mode now makes this
  fall-back path discoverable — every CAS call shows
  "Computed via SymEngine.X" in the help popover, and the
  PLAN P10 entry tracks the three paths to bringing CAS to
  the other platforms (reduced-capability web, WASM bridge,
  or remote bridge service).

**Documented but not yet built**

- **PLAN P10** — web-deployment paths (Vercel / HF Spaces /
  Cloud Run). Three options sketched with scope estimates;
  deferred until we want a "try in the browser" link.
- **Stale debt** cleared: Matrix self-test tile is now
  `kDebugMode`-gated (was already shipped in `f1d084d`;
  PLAN entry marked SHIPPED in v0.3.0 docs pass).
